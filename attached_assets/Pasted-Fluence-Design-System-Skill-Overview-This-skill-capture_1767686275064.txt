Fluence Design System Skill
Overview
This skill captures the design philosophy and patterns from Getfluence.com—a premium B2B SaaS platform for sponsored content and netlinking. The design emphasizes professional trust-building, clean data presentation, and conversion-focused layouts that work for enterprise audiences while maintaining visual sophistication.
When to Use This Skill
Apply this skill when creating:

B2B SaaS landing pages or marketing sites
Marketplace or platform interfaces
Content marketing and SEO tool websites
Enterprise software marketing pages
Professional services platforms
Any site requiring trust indicators and social proof

Core Design Philosophy
Brand Identity
Visual Language

Primary Palette: Deep professional blues, clean whites, and strategic yellow accents for CTAs
Typography Hierarchy: Bold, confident headlines with generous spacing; clean sans-serif body text
Tone: Professional yet approachable; authoritative without being corporate
Imagery: Minimalist illustrations, subtle animations, premium brand logos as social proof

Design Principles

Trust Through Transparency: Extensive use of recognizable brand logos (L'Oréal, Samsung, Uber) to establish credibility
Progressive Disclosure: Information revealed in digestible sections with clear visual hierarchy
Data-Driven Storytelling: Statistics and metrics prominently displayed as proof points
White Space Mastery: Generous spacing prevents cognitive overload
Conversion Clarity: CTAs are unmistakable with high-contrast yellow buttons

Typography Best Practices

Use system fonts or professional sans-serifs (Inter, Roboto, or similar)
Maintain consistent line-height for readability
Limit body text line length to 65-75 characters
Increase letter-spacing slightly for all-caps text
Bold weights for headings (600-700), regular for body (400)

Color Usage Guidelines

Primary Blue: Headers, primary navigation, footers, trust sections
Accent Yellow: CTAs, important highlights, conversion elements
White/Light Gray: Alternating section backgrounds for visual rhythm
Dark Gray: Body text, secondary elements
Logo Colors: Only in grayscale for consistency (use filter: grayscale(100%))

Color Psychology in B2B Context

Blue conveys trust, stability, and professionalism
Yellow creates urgency and draws attention without aggression
Generous white space communicates premium positioning
Grayscale logos prevent visual competition and maintain focus

Animation Principles

Subtle and Purposeful: Never animate for animation's sake
Performance First: Use transform and opacity for 60fps animations
Duration: 200-300ms for micro-interactions, 400-600ms for section reveals
Easing: ease-out for entrances, ease-in for exits, ease-in-out for transforms
Loading States: Always show skeleton screens or spinners during async operations

Mobile-First Considerations

Stack all multi-column layouts on mobile
Hamburger menu with full-screen overlay
Touch-friendly button sizes (minimum 44px height)
Reduce padding on mobile (50% of desktop)
Hide complex animations on mobile for performance

Content Guidelines
Headline Writing
Do's:

Lead with benefits: "Be a reference in AI answers"
Use power words: "harness," "develop," "boost," "ensure"
Include quantifiable metrics: "45,000+ premium media"
Create urgency: "Give yourself the visibility your company deserves"

Don'ts:

Avoid jargon unless industry-standard
Don't bury the value proposition
Never use clickbait tactics
Avoid passive voice

Call-to-Action Copy
Effective CTA Patterns:

"Try [Product] for free" (removes risk)
"Create an account" (clear action)
"Get your [valuable thing]" (value-focused)
"See [Product] in action" (demonstration)

Button Text Best Practices:

Use first-person when possible: "Start my free trial"
Keep it short: 2-4 words maximum
Action verbs: Get, Try, Start, Create, See
Avoid: "Click here," "Submit," "Enter"

Statistics Presentation

Always include a visual indicator (%, ×, +)
Add context: "+63% increase in customer engagement"
Use large, bold typography for the number
Position near relevant feature descriptions
Source data if making specific claims

Design Anti-Patterns to Avoid
What NOT to Do

Cluttered Hero Sections: Never include more than one primary CTA in the hero
Colorful Logo Chaos: Don't display client logos in full color—always grayscale
Generic Stock Photos: Avoid people pointing at laptops; use product screenshots or custom illustrations
Aggressive Popups: No immediate exit-intent popups; respect user journey
Auto-Playing Videos: Never auto-play videos with sound
Tiny Text: Minimum 16px for body text, 14px for small print
Low Contrast: Always meet WCAG AA standards (4.5:1 for normal text)
Infinite Scroll Without Pagination: Always provide pagination options
Hidden Navigation: Don't hide primary navigation behind mystery meat icons
Form Overwhelm: Limit initial form fields to 3-5 maximum

Final Notes
This design system excels at building trust with enterprise audiences while maintaining visual sophistication. The key is balancing professional aesthetics with clear conversion paths—every element should either build credibility or guide users toward action.
Remember: Fluence's design succeeds because it treats visitors as informed decision-makers who need evidence, not persuasion. Give them data, social proof, and a frictionless path to try the product.
When in doubt, refer back to these core principles:

Trust before promotion
Data over claims
Clarity over creativity
Action over aesthetics
Results over features
